## What Is Native Compilation?

Emacs 28+ supports ahead-of-time (AOT) compilation of Emacs Lisp to native code via [libgccjit](https://gcc.gnu.org/onlinedocs/jit/). Every `.el` file is compiled to a `.eln` native library.

Benefits:
- Startup time reduced significantly (no JIT warm-up)
- Runtime performance improvements across the board
- Consistent execution speed — no cold paths

## Configuration

Hyalo Feedstock builds Emacs with:

```bash
./configure --with-native-compilation ...
```

Pixi provides `libgccjit` in the environment. No system GCC installation required.

## Build Output

After `pixi run install`:

```
nextstep/Emacs.app/
  Contents/
    MacOS/
      Emacs             # Main executable
    Resources/
      lisp/             # .elc byte-compiled files
      native-lisp/      # .eln AOT-compiled files (architecture-specific)
```

Packages installed later by `package.el` will also be AOT-compiled on first load.

## Hyalo Integration

Hyalo's `init.el` loads the full init system after Emacs starts. Because native compilation is enabled, every `use-package` block's autoloads and deferred loads execute at native speed.

The `hyalo-feedstock` Emacs binary is the required runtime for Hyalo's Swift dynamic module (`.dylib`) — it must be built with `--with-modules`.
