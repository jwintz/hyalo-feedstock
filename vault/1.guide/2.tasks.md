All build operations are `pixi run` tasks. Tasks depend on each other in sequence.

## Core Pipeline

| Task | Description |
|------|-------------|
| `prep` | Sync `emacs/` submodule to `emacs-build-macos/` via rsync |
| `patch` | Apply patches from `patches/` to `emacs-build-macos/` |
| `autogen` | Run `./autogen.sh` inside `emacs-build-macos/` |
| `configure` | Run `./configure` with `--with-ns`, `--with-modules`, `--with-native-compilation`, and Pixi-provided dependencies |
| `build` | `make -j$(nproc)` ‚Äî compile all of Emacs |
| `install` | `make install` ‚Üí creates `nextstep/Emacs.app`, then codesigns it |
| `run` | Launch the built `Emacs.app` |

## Maintenance

| Task | Description |
|------|-------------|
| `check` | Run the Emacs test suite |
| `clean` | `make clean` inside `emacs-build-macos/` |
| `clean-builds` | Remove `emacs-build-macos/` entirely |
| `distclean` | `make distclean` inside `emacs-build-macos/` |

## Dependencies

Pixi provides all C dependencies in the environment:

- `gnutls` ‚Äî TLS support for HTTPS packages
- `libxml2` ‚Äî XML parsing
- `tree-sitter` ‚Äî tree-sitter grammar support
- `libgccjit` ‚Äî native compilation backend
- `imagemagick` ‚Äî image support
- `sqlite` ‚Äî SQLite integration

These are resolved automatically; no Homebrew or system packages needed.
